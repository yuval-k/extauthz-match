üéØ What Is This?

This is a live demo of Agentgateway / Envoy's external authorization (ext_authz) system with a Tinder-like mobile interface. When clients make requests through your API gateway, they appear as "cards" on your phone that you can swipe to approve or deny.

üíò How To Use

  1. Scan the QR code with your phone (or open the link on this device)
  2. Watch the requests appear - This page automatically sends 5 test requests
  3. Swipe right ‚ù§Ô∏è to approve a request (returns 200 OK)
  4. Swipe left ‚ùå to deny a request (returns 403 Forbidden)
  5. Watch the results update in real-time on this page!

üîê Security Features

üîí End-to-End Encrypted: All communication uses AES-256-GCM encryption. The relay server never sees your request data - only encrypted messages pass through!

  • Zero-trust architecture: Encryption key stays on your devices
  • Ephemeral keys: New key generated on each server restart
  • URL fragment security: Key never sent to servers (stays client-side)

üèóÔ∏è Architecture

The system has three main components:

  • Gateway (Agentgateway or envoy): Intercepts requests and asks for authorization
  • AuthZ Server: Handles authorization logic and encryption
  • Relay Server: Routes encrypted messages between server and mobile
  • Mobile UI: Beautiful swipe interface for approvals
